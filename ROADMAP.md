# Roadmap

Panji Online Router is intentionally small. The goal is to stay simple while making static publishing easier to maintain.

## Near term

- Improve the router homepage design.
- Add a small route creation helper.
- Add route metadata for title, description, and status.
- Add examples for reports, demos, and project previews.
- Add contributor documentation.

## Maintainer workflow

- Keep every public page under `public/[slug]/`.
- Review each new route through pull requests.
- Keep secrets and private data out of the repository.
- Use Git history as the audit trail for published pages.

## Possible future features

- Route manifest generation.
- Broken-link checking.
- Static search index.
- Simple screenshot preview workflow.
- Optional GitHub Actions validation.
